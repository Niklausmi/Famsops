FleetCRM — Setup Guide

Complete step-by-step instructions to go from downloaded files to a live system

6 HTML/JS Files 1 Apps Script 5 Google Sheet Tabs

📁 Your CRM Folder — All Files

📁 FleetCRM/

├── dashboard.html ← Main dashboard
├── sales_leads.html ← Sales leads form + table
├── vehicle_job_order.html ← 9-step job order form

⚠ Important All 6 HTML/JS files must be in the same folder. Do not rename crm-core.js.
Step 1 — Google Sheet
1 Create & configure your Google Sheet

Open sheets.google.com and create a new spreadsheet. Name it anything (e.g. FleetCRM Data).

Then go to Extensions → Apps Script, delete the default code, and paste the entire contents of Code.gs.

In the Apps Script editor, select the setupAllSheets function from the dropdown and click ▶ Run. This automatically creates all 5 sheet tabs with the correct headers in one click.

✓ Tabs created by setupAllSheets() Sales Leads  ·  Job Orders  ·  Trackers  ·  SIMs  ·  Assignments
⚠ If you already have tabs Make sure your existing tab names match exactly (case-sensitive). You can change them in the TABS object at the top of Code.gs.
Step 2 — Deploy Apps Script
2 Publish as a Web App

In the Apps Script editor:

  • Click Deploy → New deployment
  • Click the ⚙ gear icon → select Web app
  • Set Execute as: Me
  • Set Who has access: Anyone
  • Click Deploy and copy the URL shown

The URL looks like:

https://script.google.com/macros/s/AKfycb.......XXXX/exec
⚠ Every time you edit Code.gs You must create a New Deployment (not edit existing) to push changes live. The URL changes each time — update crm-core.js accordingly.
Step 3 — Connect the URL
3 Paste your URL into crm-core.js

Open crm-core.js in any text editor (Notepad, VS Code, etc.) and replace the placeholder on line 6:

// ✏️ PASTE YOUR GOOGLE APPS SCRIPT URL HERE
const CRM_SCRIPT_URL = "https://script.google.com/macros/s/YOUR_ID/exec";

Save the file. That's the only file you need to edit. All 5 HTML pages read the URL from crm-core.js automatically.

Step 4 — Run Locally
4 Start your local server

Open a terminal / command prompt in the FleetCRM folder and run:

python -m http.server 8080

Then open your browser and go to:

http://localhost:8080/index.html
💡 Tip — open the right folder first cd "C:\path\to\your\FleetCRM folder" then run the python command. The folder shown before > in your terminal must be the same folder as your HTML files.

Troubleshooting
8 Common issues and fixes
ProblemFix
404 on localhost Run cd into the exact folder containing your HTML files before python -m http.server 8080
Data not appearing in Sheet Make sure you re-deployed after editing Code.gs. Check the URL in crm-core.js matches the latest deployment URL.
Sheet tab not found error Run setupAllSheets() from the Apps Script editor, or check tab names match the TABS object in Code.gs exactly.
Redirected to login on every page The browser must support sessionStorage. Ensure you're accessing via http://localhost:8080 not via file:///.
Inventory table won't load GET requests only work if the Apps Script deployment is set to Who has access: Anyone. Check your deployment settings.
Can't assign SIM — dropdowns empty Click Refresh on both the Trackers and SIMs tabs first so the data is loaded into memory before opening the Assign modal.
Go-Live Checklist
  • All 7 files are in the same folder (index.html, dashboard.html, sales_leads.html, vehicle_job_order.html, inventory.html, crm-core.js, Code.gs)
  • Code.gs pasted into Apps Script and saved
  • setupAllSheets() run — 5 tabs created in Google Sheet
  • Web App deployed — Execute as: Me, Access: Anyone
  • CRM_SCRIPT_URL updated in crm-core.js with your deployment URL
  • Python server running from the correct folder
  • Opened via http://localhost:8080/index.html (not file:///)
  • Test login with admin / admin123
  • Test Stock In — add a Tracker, verify row appears in Google Sheet
  • Test Lead form — submit a lead, verify row appears in Sales Leads tab
